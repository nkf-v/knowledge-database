- Application
	Бизнесовая логика
	Может использовать:
	- Domain
	- _Vendor_ (не желательно)
	
- Infrastructure
	Техническая часть которая автоматизирует бизнесовые интерфейсы скрывая реализацию от Application
	Может использовать
	- Domain
	- Application
	- Vendor
	- Presenter
	
- Presenter
	Уровень описывающий данные в необходмом виде
	Может использовать
	- Domain
	- Application
	- Vendor
	- Infrastructure
	
- Domain
	Бизнесовые классы описывающие ключевые элементы бизнесовой логики. Должен быть максимально независимым и описывать реальную бизнесовую сущность.
	Ни от кого не зависит

- Vendor
	Классы библиотек ни от кого не зависят.
